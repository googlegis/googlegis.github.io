为什么TypeScript现在和将来都会更加流行！

发表于 2015-05-22 | 分类于 Translation |

当前对于JavaScript开发者来说，是一个空前的好机遇！从一个网站的前端到后台Javascript已经是无处不
在，甚至于很多桌面版本的文本编辑器都是由Javascript驱动的。事实上，你现在可以用web技术来实现一个
桌面应用程序的全部功能。在你的JavaScript技术越来越熟练的情况下，最重要的一点就是保证你的代码在你
的掌控之中，免得变成一堆错综复杂的玩意。

##TypeScript

TypeScript 遵守 ECMAScript 6(ES6) 中的可选类型规范。

###ECMASCRIPT6

ECMASCript 是JavaScript语言的标准规范，
有时被称为ECMAScript Harmony或者ES.再者，当前的JavaScript的规范是ECMAScript5。 仅有几个
浏览器支持ES6规范，但是支持的浏览器肯定会越来越多。既然ES6是默认兼容ES5的语法，那么你就可以不需要
改动任何代码就能在ES6下面运行。然而最重要的一点就是，学习TypeScript语法就是在学习ES6语法，所以
你是在学习将来的JavaScript语法，而不是在学习一门完全新的语法，比如CoffeeScript

###可选类型
TypeScript 自己的可选类型。 类型语言如Java,C#,Object-C 需要你在初始化变量时指定类型。
Java中Srtring name =”Andrew”,C#中 string name =”Andrew”; 而在 Object-C中，
NSString name = “Andrew”.

而在TypeScript中，可以定义为：var name:string = “Andrew”. 而作为可选的，也可以声明为
var name = “Andrew”. 在严格类型语言中，你必须时刻定义变量类型，在TypeScript中，则不需要。

在代码中定义变量类型有助于文本编辑器和IDE在不需要编译代码的情况下就能快速给出智能提示。

同样，它也可以帮你自动完成代码。有些文本编辑器是根据你的项目文件，它不可能知道你所有变量的类型，但是
当你调用函数时，传递了一个类似名称的变量，其实是另外一个对应的类型。但是在TypeScript中，在调用函数
时，代码编辑器能够更加智能提示合适的变量。这个效率非常高，相当于是代码在写代码。

而且还减少了在方法调用或者方法调用返回值的时候，前后频繁查看文档查找各个变量注释的麻烦。

TypeScript无处不在

凭借简洁的ES6语法，智能代码完成和可选类型提示，TypeScript极大提高了效率，并且已经被应用在许多大型
的项目和框架上如：AngularJS 2.0 和 Ionic Framework 2.0.

在你的JavaScript项目还没有开始之前，你可以记得给TypeScript留有一席之地。TypeScript可以在各种
环境下从浏览器到Node.js都可以帮你提高效率。当你编写代码时，可以获得自动完成和提示。

Typeing 正在成为你身边的一个编程语言

有些程序员喜欢动态类型，但是你不需要在你写代码的时候声明一个变量，主要的一个原因是代码不长。
然而，这似乎是一个趋势，很多编程语言汇聚了很多功能，从generics，固定变量，异步编程，可选类型甚至
智能编译。甚至Ruby之父-Yukihiro Matsumoto说：静态类型将要到达版本3.0.

在代码中书写类型不只帮助提高编写效率，还能在程序运行时帮助解释和编译代码。

下一步

打开 TypeScript官网,查看一下主页介绍，当然也可以在交互式Playground中测试TypeScript。
也可以下载跨平台的智能代码提示文本编辑器-Visual Studio Code.你也可以查看我们的 JavaScript
路线图 中的TypeScript。

总结

正因为TypeScript的清晰的ES6语法和可选类型，而不是一个完全不同的语法，它不但提高了工作效率还可能
成为JavaScript的未来和趋势。TypeScript的流行来自于一些流行框架的使用比如AngularJS.
可选类型也可能是你最喜欢的动态类型语言的未来和趋势。谁知道，TypeScript会不会成为JavaScript的一个
新的版本呢。现在不只是一个学习JavaScript的机会，也是一个了解TypeScript的好机会。
