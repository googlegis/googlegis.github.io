写教程是真难！

发表于 2015-07-26 分类于 Swift Disqus：

第二章终于写完了，原本打算 3 天写完，结果实际上花费了 7 天时间。

一般我的步骤是按照英文教程先编一遍应用，出来一个可以跑的应用。然后再

按照教程，再来一遍代码，在代码的过程中截图、写作，发现重心全放到文章的梳理和

协调上去了，根本对编程内容和技巧没有一点提高。

所以，改变了方法，第一遍还是按照英文教程编一个应用，

第二遍是细读，根据读得内容写文章，这个过程不牵涉到写代码和截图。

第三遍是写代码和截图。

这样的话，三遍下来，一章教程 300 页，第一遍基本是 6-7天，第二遍4-5天，第三遍基本是10天。

也就是说，300 页的教程要20天才能完成，现在真的觉得那些用心写教程，写书的人不容易。

在这方面做的比较牛逼的一个是李笑来，一个是阮一峰，他们的文章和出版的书，在他们的博客里都

有，你当然可以不花钱就能够读到。现在我觉得，如果觉得别人的劳动对你有所帮助，有价值，

还是掏钱买正版书籍看吧，一来养眼益目，二来是对别人劳动的尊重。

在第二章中，后半段纯粹就是应付差事了，把代码拿来输入一遍，也没有过脑子，也没有带 Xcode 7

中测试，需要 swift 2 版本的代码到 github 上下吧。

https://github.com/googlegis/iOSApprentice
