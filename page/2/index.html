添加系统引用和包的引用

发表于 2016-07-08 | 分类于 Xamarin |

本文接着讲如何添加 dll 库和引用包，开始的时候，我看项目里有

refrence 和 packages 两个部分，搞不清两者关系，后来从添加的内容看，

refrence 中添加的都是和 .Net 有关的 dll 库，packages 里添加的是

和网页开发相关的 js 库(当然也包括一部分和后台 C# 相关的 dll 文件)。

后面一篇专门讲 log4net 的文章时会有提到！

要添加系统库，需要在 Refrences 文件夹上右键，选择 Edit Refrences…

或者直接双击 Refrences 文件夹。

然后选择自己需要的库，这里添加最常见的 System.Data 库，

因为你要保存查询出来的数据库结果集，这个使用非常方便。 打勾-OK！

然后就可以看到 Refrences 中的 System.Data 引用。

在.cs 文件中，添加 using 。然后在代码中就可以直接用 DataTable class

进行代码处理了。

添加包的引用和 Refrences 差不多，文件夹上右键，Add Packages… 或者双击，

然后弹出添加引用的窗口，这里是使用官方 NuGet Gallery。 如果你够仔细的话，可以

在配置中找到包源的配置界面，可以自己添加包源配置。

比如，我们搜索 jQuery 。会出来 jQuery 库， 和 jQuery UI 库，我们选中

jQuery UI 库， 能够看到在右侧的说明中，有 Dependencies 依赖， jQuery(≥1.6)。

直接勾选 jQuery UI，点击 Add Package，然后能够看到正在安装的过程。

安装好之后，就能在 Packages 中看到 jQuery 和 jQuery UI。

这说明不需要担心你选择的库的依赖，直接点击安装，其它的事情，IDE 来帮你做。

然后我们打开项目中的 packages.config 文件，来看一下我们项目中 yij

引用的类库和包！

打开 Scripts 文件夹，能够看到引用 jQuery 的文件都在这里了。

这里需要严重吐槽一下，这种依赖添加只会按照最小版本进行添加，这里添加

进来的 jQuery 的版本就是 1.6.4 ，所以如果想要使用更新版本的 jQuery，

还需要自己手动添加。

但是还有另外一点支持不好的地方，每个库都只能引用 NuGet 中最新的版本，

比如这里的 jQuery 库是 1.6.4 版本，太旧了，想更新到新版本，就会直接更新

到最新版本，而现在 NuGet 上最新的版本是 3.0.0, 而且其它依赖 jQuery 的

库的版本还没有来得及更新到 3.0.0 的版本，所以这里就又是一个坑。

需要自己手动下载 jQuery 库，进行添加引用。 我想如果添加库时，能够有一个

版本选择的功能的话，是不是方便很多。

的研究过程都记录下来，备忘和分享！

.Net 的跨平台开发，现在无非是两种方式， 一种是 Mono SDK, 一种是

Asp.Net Core. 但是 Asp.Net Core 还是太新，只能作为对比，还不敢拿

来直接作为开发使用。所以，默认使用 Mono。

Mono 是一个跨平台的 .Net 解决方案，最后被微软收购并进行维护和支持。

应该说还是有一定的群众基础的。

在 Linux 系统下推荐的是 MonoDeveloper

http://www.monodevelop.com

在 Windows 和 Mac 系统下推荐的是 Xamarin Studio。

http://www.monodevelop.com/download/

从 Mono-Develop 的官方页面下载的版本是 5.6

但是从 Xamarin 的官方页面下载的是 6.0

https://store.xamarin.com/account/my/subscription/downloads

如果只是开发 Web 页面，只下载 Xamarin Studio 的安装包即可。

然后直接双击安装， 如果没有安装 Mono SDK，会提示要先安装 SDK。当然也可以

从网站先下载安装 Mono SDK，安装再安装 Xamarin Studio。

http://www.mono-project.com/download/

打开 Xamarin Studio，新建 Asp.Net Mvc 项目进行测试。

至此，说明 Xamarin Studio 安装成功！

写教程是真难！

发表于 2015-07-26 | 分类于 Swift |

第二章终于写完了，原本打算 3 天写完，结果实际上花费了 7 天时间。

一般我的步骤是按照英文教程先编一遍应用，出来一个可以跑的应用。然后再

按照教程，再来一遍代码，在代码的过程中截图、写作，发现重心全放到文章的梳理和

协调上去了，根本对编程内容和技巧没有一点提高。

所以，改变了方法，第一遍还是按照英文教程编一个应用，

第二遍是细读，根据读得内容写文章，这个过程不牵涉到写代码和截图。

第三遍是写代码和截图。

这样的话，三遍下来，一章教程 300 页，第一遍基本是 6-7天，第二遍4-5天，第三遍基本是10天。

也就是说，300 页的教程要20天才能完成，现在真的觉得那些用心写教程，写书的人不容易。

在这方面做的比较牛逼的一个是李笑来，一个是阮一峰，他们的文章和出版的书，在他们的博客里都

有，你当然可以不花钱就能够读到。现在我觉得，如果觉得别人的劳动对你有所帮助，有价值，

还是掏钱买正版书籍看吧，一来养眼益目，二来是对别人劳动的尊重。

在第二章中，后半段纯粹就是应付差事了，把代码拿来输入一遍，也没有过脑子，也没有带 Xcode 7
