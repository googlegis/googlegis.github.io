刻苦 成功 自信

发表于 2016-09-27 | 分类于 育儿经 |

刻苦努力才能取得成功，有了成功的资本，人生才能更自信！

这本来是一个通俗易懂，童叟无欺，放之四海而皆准的道理。但有的时候，不但孩

童做不到，就是大人也做不到！！！

上周四，上幼儿园中班的娃娃回来说，老师让家长拿着小孩的手画五角星。所以我

先拿了张纸告诉她怎么画，她学的很快，练习了5-6个之后，她就画的有模有样了，

虽然没有达到横平竖直，角度相同，但从形状和笔划能够看出五角星的样式和工整

了。

所以很满意，就夸奖她画的认真，而且很好看！ 受到夸奖的小孩子很容易表现的，

然后自己又马不停蹄的画了很多个，每个画的都很不错！每画一个就拿过来让看是

否画的好，意思就是你再表扬我一下呗！！！

连着画了几个工整五角星之后，我说帮你找张纸，你自己画，然后明天交给老师。

前面都画的那么好，我就没有看着她画，她还是每次画一个，然后再过来让看，

前面两个画的还不错，可是后面的画的越来越差，不但笔划顺序错了，连基本的

直线都画不好了，基本就像是一个手枪了。五角星能画成手枪，也算是潦草到家

了。

她画了6个拿过来的时候，我看后面几个已经一塌糊涂了，所以假装生气的说，

这几个肯定是你没有用心画，你刚才自己画着玩的时候都能画好，现在却画成

这样，说明你没有用心认真的画，我再给你重新拿张纸，再重新画。

她很委屈，可是自己画的确实不好，只好再画，我停留了一小会去看她，

第一个就画成了手枪，我严厉的告诉她，这样肯定是不对的，你是不是忘了怎

么画了，我又找来一张纸，和她一起画，我画一个，让她看着，然后让她自己

再画一个，我再画一个，让她看着，然后再让她画一个。等她学的像模像样了，

就让她自己画。练习了4-5个之后，让她再重新画了一张，总共画了7个，前面

4个画的还可以，后面3个的线条稍微有点弯曲，假装生气的稍微指点了一下，

放到书包里让她明天带到学校交给老师！！！

我本来只是想锻炼她的耐心，注意力和认真程度，没有想到却带给她莫大的自信。

第二天，老师让小朋友自己画五角星，她自己在幼儿园里画的又快又好，很快就

完成了，还受到了老师的表扬。然后老师告诉她说，去看一下哪个小朋友没有画

完或者不会画，你去帮她画，据她自己说，她在班里问，谁需要帮忙画五角星？

然后有很多小朋友举手，她就帮了好多小朋友画五角星。当然了还是受到了老师

的表扬。

下午从学校练完舞蹈回来，回来后，她兴高采烈的绘声绘色的和我描述了一番，

休息了一下，就说要学习，结果那天晚上停电了，刚开始的时候没有找到蜡烛，

所以一直告诉她等来电了再学习，最后捣鼓出来一小截蜡烛，刚点亮，她就找

来一张纸，说是要画五角星。

哈哈！！！ 家里每回来一个人，她都要拿着画的五角星让家人评价她画的是否好。

言语中充满了自信和激情。这也是我想要的。

鼓励+表扬–>努力+成功–>自信

看来在教育孩童的这条路上，我也是边学习，边摸索，以后的路还很长！

添加系统引用和包的引用

发表于 2016-07-08 | 分类于 Xamarin |

本文接着讲如何添加 dll 库和引用包，开始的时候，我看项目里有

refrence 和 packages 两个部分，搞不清两者关系，后来从添加的内容看，

refrence 中添加的都是和 .Net 有关的 dll 库，packages 里添加的是

和网页开发相关的 js 库(当然也包括一部分和后台 C# 相关的 dll 文件)。

后面一篇专门讲 log4net 的文章时会有提到！

要添加系统库，需要在 Refrences 文件夹上右键，选择 Edit Refrences…

或者直接双击 Refrences 文件夹。

然后选择自己需要的库，这里添加最常见的 System.Data 库，

因为你要保存查询出来的数据库结果集，这个使用非常方便。 打勾-OK！

然后就可以看到 Refrences 中的 System.Data 引用。

在.cs 文件中，添加 using 。然后在代码中就可以直接用 DataTable class

进行代码处理了。

添加包的引用和 Refrences 差不多，文件夹上右键，Add Packages… 或者双击，

然后弹出添加引用的窗口，这里是使用官方 NuGet Gallery。 如果你够仔细的话，可以

在配置中找到包源的配置界面，可以自己添加包源配置。

比如，我们搜索 jQuery 。会出来 jQuery 库， 和 jQuery UI 库，我们选中

jQuery UI 库， 能够看到在右侧的说明中，有 Dependencies 依赖， jQuery(≥1.6)。

直接勾选 jQuery UI，点击 Add Package，然后能够看到正在安装的过程。

安装好之后，就能在 Packages 中看到 jQuery 和 jQuery UI。

这说明不需要担心你选择的库的依赖，直接点击安装，其它的事情，IDE 来帮你做。

然后我们打开项目中的 packages.config 文件，来看一下我们项目中 yij

引用的类库和包！

打开 Scripts 文件夹，能够看到引用 jQuery 的文件都在这里了。

这里需要严重吐槽一下，这种依赖添加只会按照最小版本进行添加，这里添加

进来的 jQuery 的版本就是 1.6.4 ，所以如果想要使用更新版本的 jQuery，

还需要自己手动添加。

但是还有另外一点支持不好的地方，每个库都只能引用 NuGet 中最新的版本，

比如这里的 jQuery 库是 1.6.4 版本，太旧了，想更新到新版本，就会直接更新

到最新版本，而现在 NuGet 上最新的版本是 3.0.0, 而且其它依赖 jQuery 的

库的版本还没有来得及更新到 3.0.0 的版本，所以这里就又是一个坑。

需要自己手动下载 jQuery 库，进行添加引用。 我想如果添加库时，能够有一个

版本选择的功能的话，是不是方便很多。
