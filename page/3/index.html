王的男人之秘密服务

王的男人-秘密服务，这个是我最近看到的最逗比的翻译，可是这个翻译可是正规的公司翻译的。

这个片子也是因为听说好看才去看的，也是传统的故事情节，小人物的成长史。

一个足够聪明和足够有能力的小混混，因为某些原因不得志，整日混来混去，幸得一位极好

的老师指点，成为一代人才并最终力挽狂澜的故事。

这样的故事情节被周星驰老师不知道炒了多少剩饭了，国内观众应该比较认可这样的故事套路，

稍加处理就可以很容易得到好评，但是整个故事看完，就记住了男主角的师傅在教堂大开杀戒，

杀的昏天暗地，神哭鬼号的那段，和结尾瑞典公主的大屁股。

估计也只有瑞典的公主可以被影片拿来调戏吧。

不过教堂杀戮那段拍的实在是太过瘾了，死的太惨太真实了，一个个活人在你眼前就那么眼睁睁的被

杀死了，还都能看到被砍的位置，我想应该是位中国的武指设计的吧。国外的动作设计根本设计不了

这么精细的动作和场面。

如果非要王的男人和疯狂的max有个比较的话，但就导演功力上，王的男人要高疯狂的max一头。

疯狂的max是用机械的轰鸣和躁动的电吉他，配上模特的身材刺激你的肾上腺，让你无暇思考它的故事

情节啊，故事推动啊，合理性啊。 这要是在中国，被一个中国导演拍出个这样的片子，肯定被批个

傻大粗装(如姜文);而王的男人则更注重细节，情节推动，故事的叙述性和完整性(再如姜文)。

单个打斗场面细腻而清晰，是人物推动故事发展，而不是导演推动故事发展。

虽然我找了疯狂的max和王的男人这么多缺点，但不可否认它们还都是好电影，只不过不是那种让人

能记住的电影。

疯狂的max导演就是要不走寻常路，王的男人导演是要突破007的极限。

现在，你有一个闪闪发亮的新Mac(二手Mac你擦干净的也算新的)摆在桌子上了，现在开始安装一些

用来iOS开发的重要的软件了-XCode。 XCode 是你开发iOS 应用的开发环境(IDE-integrated development

environment)。你可以从 Mac 应用商店免费下载，或许你现在正在下载。

在Xcode下，你可以编写代码，调试代码，绘制界面，单元测试以及很多很多其他的事情，你也可以

使用Xcode上传你的app到应用商店。掌握并熟悉Xcode，这是每个iOS开发者最重要的软件。

3. 学习编程基础(可能是最难的部分)

现在是最难得部分了，你需要直接编写代码。如果你有其他的编程背景，学些Objective-C(较难)和

Swift(较容易)不会太难，它们是标准的，面向对象的编程语言。如果你之前的人生中都没有编写过

一句代码，别担心，这里有两个针对初学者的很不错的资源网站。

1.Ry’s Objective-C tutorial–Objective-C 粉丝的老学校，你不必真的去完全掌握

Objective-C(Swift才是王道),至少要懂得基本语法，能够读写代码。

2.Swift language guide 这是我们要学的Swift语言的官方文档，Apple出品，必属精品。

你不必对所有的知识点都有深度的了解，这些会在以后的实际工作经验中接触到，但是你要把自己完全

沉浸在变量、指针、类、数据类型和递归这些知识的海洋里，它提供给你更进一步学习的环境。

4. 根据教程一步一步写一些不同的app

当你想要做些什么有用的app时，这些网站也许能帮助你。

1.AppCoda-比较适合初学者，你将会发现很多不同的教程，跟着说明一步一步来，

确保能够完整的走一遍。

2.Ray Wdnderlick–另外一个有着海量iOS开发教程的网站，一步一步学习是最重要的。

但是不要只在这些网上教程上来回学习，自己亲自动手做一些app，比如计算器，天气，货币转换，

音乐app，任何你可以在教程上找到的都可以做。

跟着这些app教程学习，直到你觉得对Xcode 和 编程语言(Swift) 比较熟悉了。就要开始下面的步骤。

###5. 编写自己的app

现在是重点部分了，你需要做一个自己的app将来可以在面试的时候给你未来的雇主演示。

但是，别紧张，你不必做再做一个Facebook出来，你只是要找一个低级职位，OK？一个你需要从竞争

对手那里学习，而不是需要你一个需要5年经验的专家。

所以，冷静下来想一下，你要开发在哪个行业或者领域的app.

可能你写的是一个使用了网络传输的app，也可能是你使用UIKit设计了一个非常舒服的复杂用户接口。

或者可能你最擅长的开发是一个音乐或声音播放器。使用这些有趣的知识创建自己的app，主要要一个

干净、清晰、能够正常工作的代码。

###6. 同时，尽可能多的学习软件开发技能

如果你读到这里，你可能不希望去大学再花费几年时间学习计算机科学。好消息是你不必这么做。

很多计算机科学/编程/软件科学 可能都能在家里在线学习。

当然，这样肯定是拿不到学位的，但是获得一个iOS开发者的工作不需要这个，参看上面的图片，学习

这些知识点。我不会给出链接–搜索信息是一个开发者的技巧中最重要的一点。

开始学习吧，Google 是你的朋友。

###7. 完成你的app

在经历了几天，几周，甚至几个月的学习和工作在你的app上，你应该有一个完整的app了，这个app就

是你的简历，你应该把它当成你最好的项目，这里有一些公司主要看中的几个点：

  • 可以运行的app

  • 干净的代码

  • 代码组织–类的粒度，变量合适的名字，Xcode项目中合适的文件分组。

  • storyboard的使用（如果你能同时使用storyboard和代码进行用户界面编写会加分）

  • CocoaPds 使用。

  • 简单的单元测试。

  • 扩展库的使用（比如Github上的一些开源库，实际工作中确实经常用到这些库。）

当然，所有的一切都取决公司和你工作需要的内容，但是这些技能基本就能让你适应基本的初级iOS

开发人员职位。

OK，现在你有一个漂亮，闪闪发亮的app了，下一步骤是：

###8. 把你的app发布到商店

OK，首先必须要声明一下：这一步是可选的，因为要做这一步，你必须需要一个每年99$的 iOS 开发

账号，而这笔花费对你的app将是一个大的挑战。

你有没有app在应用商店上架，是大多数公司给你简历加分的重要标准。

在应用商店中有app说明你懂得发布应用的流程，Apple 对app的限制是什么？除了app本身还需要什么
（说明，关键字，截图，说明视频）等等。

你不必非得做这个步骤，但是我强烈建议你完成。(我在应用商店的app帮助我找到了第一个工作。)

###9. 上传你的app到Github

GitHub 是一个以代码分享为主的社交平台(第二貌似是Bitbucket)

在这里，你可以上传你的代码，可以查看别人的代码，你也可以给开源项目贡献代码。它是如此广泛

的被使用，甚至如果你是个人开发者也能从中获益。更好的组织代码和更好的备份代码。

为什么要上传代码？好吧，是为了给你未来的雇主查看。

千万不要用Email发送代码，我们又不是在90年代。

###10. 联系你想工作的公司

现在你准备开始作为iOS开发者的第一份工作了，重要的是你已经有了获取第一份工作的技能。

当你做到这一点时，剩下的就看命运了。

####所以，准备好简历，找一家好公司，去开发一些牛逼的app吧。

假如你现在有了开发技能，在你开始找工作之前，想作为一个个人开发者，但是不知道想做什么样

的app，想要寻找一些灵感，来订阅我的iOS应用和游戏新闻吧。

主要是我打理，这里有iOS开发者的各种app。

如果已经在应用商店里有了非常酷的app，你可以在 Twitter 上给我发消息，

或许下次的推荐里就会有你的app出现。

选中上下左右四个方向的 I 红线，并分别设置数值为 0. 取消 Constrain to margins 勾选

同样的操作对于 aboutViewController 中的 background。
