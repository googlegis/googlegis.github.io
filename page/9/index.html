使用Swift和GoogleMapiOSSDK开发地图应用示例(1)

发表于 2015-05-28 | 分类于 Swift |

iOS应用中的地图类别中，开发者可以有千万种想法和功能在上面实现，这里所要介绍的是几个基本功能：显示当
前位置，规划旅游路线(支持自定义中转点), 甚至从另外一个完全不同的方法去探索地图各种创意和功能, 毫无
疑问所有这些都会积累经验带来意想不到的结果。

iOS从开始一直到版本5.1(包括)，iOS所有的地图应用及相关服务都是使用Google地图服务。但是，自从Apple
推出了自己一套全新的地图框架Map Kit后，就再也没有Google Map 什么事了。MapKit一直使用到现在的
iOS8.3。当Apple决定放弃使用GoogleMapService时，Google决定自己推出一套自己的跨平台的MapsSDK，
当然也包括iOS，来和iOS上的MapKit以及其它平台上的mapSDKs进行竞争。直到现在，Google已然是这方面
的佼佼者,大量的开发者都在使用GoogleMapSDK。所以，写这篇iOS平台上GoogleMapsSDK的介绍还是有一定
价值的。

现在iOS版本GoogleMapsSDK的版本号1.9.2,它包含了很多在web版本中的功能，web版本也有很多功能是
iOS版本中没有的。需要指出的一点就是在这个版本里，SDK包非常大，所以如果需要把框架源码文件拷贝到你的
项目里的话就要先考虑一下了。然后，功能的兴趣点和重要性又使得它成为了开发者的不二之选。

相对于原作者的其它教程,前面介绍的篇幅相对
较短，那是因为后面我们有大量的东西需要介绍。在介绍部分长篇大论的讨论各种功能就没有意义了,我们会在后
面进行详述。我想说的就是我们要做好遇到一些真正有兴趣的事情。如果你过去没有使用过GoogleMapsSDK进行
过开发，你绝对会在学习的过程中非常享受。下面我总结一下使用GoogleMapsSDK开发地图应用的几个知识点：

显示使用者当前的实时位置

如何显示一个地址所在位置点

如何显示规划路径

为什么TypeScript现在和将来都会更加流行！

发表于 2015-05-22 | 分类于 Translation |

当前对于JavaScript开发者来说，是一个空前的好机遇！从一个网站的前端到后台Javascript已经是无处不
在，甚至于很多桌面版本的文本编辑器都是由Javascript驱动的。事实上，你现在可以用web技术来实现一个
桌面应用程序的全部功能。在你的JavaScript技术越来越熟练的情况下，最重要的一点就是保证你的代码在你
的掌控之中，免得变成一堆错综复杂的玩意。

##TypeScript

TypeScript 遵守 ECMAScript 6(ES6) 中的可选类型规范。

###ECMASCRIPT6

ECMASCript 是JavaScript语言的标准规范，
有时被称为ECMAScript Harmony或者ES.再者，当前的JavaScript的规范是ECMAScript5。 仅有几个
浏览器支持ES6规范，但是支持的浏览器肯定会越来越多。既然ES6是默认兼容ES5的语法，那么你就可以不需要
改动任何代码就能在ES6下面运行。然而最重要的一点就是，学习TypeScript语法就是在学习ES6语法，所以
你是在学习将来的JavaScript语法，而不是在学习一门完全新的语法，比如CoffeeScript

###可选类型
TypeScript 自己的可选类型。 类型语言如Java,C#,Object-C 需要你在初始化变量时指定类型。
Java中Srtring name =”Andrew”,C#中 string name =”Andrew”; 而在 Object-C中，
NSString name = “Andrew”.

而在TypeScript中，可以定义为：var name:string = “Andrew”. 而作为可选的，也可以声明为
var name = “Andrew”. 在严格类型语言中，你必须时刻定义变量类型，在TypeScript中，则不需要。

在代码中定义变量类型有助于文本编辑器和IDE在不需要编译代码的情况下就能快速给出智能提示。

同样，它也可以帮你自动完成代码。有些文本编辑器是根据你的项目文件，它不可能知道你所有变量的类型，但是
当你调用函数时，传递了一个类似名称的变量，其实是另外一个对应的类型。但是在TypeScript中，在调用函数
时，代码编辑器能够更加智能提示合适的变量。这个效率非常高，相当于是代码在写代码。

而且还减少了在方法调用或者方法调用返回值的时候，前后频繁查看文档查找各个变量注释的麻烦。

TypeScript无处不在

凭借简洁的ES6语法，智能代码完成和可选类型提示，TypeScript极大提高了效率，并且已经被应用在许多大型
的项目和框架上如：AngularJS 2.0 和 Ionic Framework 2.0.

在你的JavaScript项目还没有开始之前，你可以记得给TypeScript留有一席之地。TypeScript可以在各种
环境下从浏览器到Node.js都可以帮你提高效率。当你编写代码时，可以获得自动完成和提示。

Typeing 正在成为你身边的一个编程语言

有些程序员喜欢动态类型，但是你不需要在你写代码的时候声明一个变量，主要的一个原因是代码不长。
然而，这似乎是一个趋势，很多编程语言汇聚了很多功能，从generics，固定变量，异步编程，可选类型甚至
智能编译。甚至Ruby之父-Yukihiro Matsumoto说：静态类型将要到达版本3.0.

在代码中书写类型不只帮助提高编写效率，还能在程序运行时帮助解释和编译代码。

下一步

打开 TypeScript官网,查看一下主页介绍，当然也可以在交互式Playground中测试TypeScript。
也可以下载跨平台的智能代码提示文本编辑器-Visual Studio Code.你也可以查看我们的 JavaScript
路线图 中的TypeScript。

总结

正因为TypeScript的清晰的ES6语法和可选类型，而不是一个完全不同的语法，它不但提高了工作效率还可能
成为JavaScript的未来和趋势。TypeScript的流行来自于一些流行框架的使用比如AngularJS.
可选类型也可能是你最喜欢的动态类型语言的未来和趋势。谁知道，TypeScript会不会成为JavaScript的一个
新的版本呢。现在不只是一个学习JavaScript的机会，也是一个了解TypeScript的好机会。

鸡毛蒜皮

发表于 2015-04-12 | 分类于 文章 |

娃子回合肥的时候，上海就我一个人了，每天的上班，学习，吃饭，睡觉，把每个时间点都充实起来。

以前娃子在身边的时候，总是感觉她闹，她哭，一点也没有时间学习。

现在她不在身边了，又觉得她会不会不开心，会不会心里有阴影，每次聊天她哭的时候，就会感觉欠了她很多。

现在的目标就是好好学习，赶快回合肥找工作，把家庭和工作都稳定下来。

老婆回合肥了，老妈回老家了。这里只剩下我一个人，每天坚持早晨自己做饭吃，下班回来自己做饭吃。一来是对外面的食物不习惯，二来是年龄大了，肠胃还是要伺候舒服点好。

以前去超市买菜，就是在那里迫不及待的催老婆，快点，快点，再快点。
不要把生命浪费在购买东西的思考上。

现在自己去超市买东西了，渐渐的对价格熟悉了，竟然也会在买菜的时候，对比一下，看一下今天的价格是多少，是不是值得买，看今天有没有特价蔬菜，今天哪些菜便宜先买回去。

今天去家乐福买菜，就买了两样0.98元／斤的蔬菜，大白菜／青萝卜，还买了特价的苹果，在那里和超市大妈抢了半天，买了4斤，回来慢慢吃。

现在我觉得这不是个性问题，这是生活习惯问题。

当你有了生活习惯后，你会自动的去做某些事情。

从超市出来的时候，一个大妈跑过来问我，你的大白菜多少钱一斤？

九毛八

大妈回头对老伴喊道，我就说九毛八比较便宜吧！

语气中满是埋怨。

多可爱的两个老人啊。

我想等我和老婆老的时候，是不是也是这样呢。

因为一点点的问题而互相埋怨呢，

我想这种埋怨不是争吵，而是对双方的极度信赖和谦让。

年逾古稀的老人肯定不会因为鸡毛蒜皮的事情而去大动干戈。

有时候生活就是要互相一点争吵才会更融洽。

以前我总是以为老爸和老妈之间相敬如宾，你侬我侬才算恩爱，现在我明白，
每个家庭每对夫妻都有自己的生活习惯和交互方式，不是每个人都适合每天都是甜甜蜜蜜的，有的时候生活也需要一点调味剂。

不亏是啥？

发表于 2015-02-12 | 分类于 育儿经 |

老家方言里有个“不亏” ，大概意思就是活该，应得的意思。一般大人训小孩都用这个，一边打一边问：亏不亏？

昨天给娃子生动的上了一节关于亏不理论实践课。

和娃子在床上玩，玩着玩着她竟然吃自己的手指头，因为她以前有个吃手的毛病给她治掉了。所以，我打了她的手一下，她又

吃，我又打，她再吃，我打，她吃。

然后我说，那我打脸了，她没有回答还继续吃，我向她脸上拍了一下，她继续吃，我又打了下很的，这下终于有反应了，( ⊙o⊙ )哇

的一下哭了。

我问：咋回事，

娃指着脸答：这里疼

我怒答：不亏。

娃边哭边问：不亏是个啥呀？

慢慢把娃哄下来，给她讲不亏是个啥。

就是你在犯错误，比如吃手，爸爸说了不让你吃，你不听，还吃，爸爸再打你，就属于不亏。

她貌似懂了。

晚上吃饭的时候，她给妈妈讲：爸爸不让吃手还吃，就是不亏。

最近，如果有什么犯错的举动的时候，我一说她，她就该说了，还有不亏。

对，就是这个意思。
