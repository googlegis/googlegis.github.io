The iOS Apprentice:Getting Started.1:快速建立应用模型

发表于 2014-10-16 分类于 Swift

最近跟随 @MacTalk 购买了两本英文版的iOS教材。完全英文版的，以前都没有想到过在自己可以完整的看完一部英文教材。

深有几点感触：

  1. 有些事情不是你不会，你不能，而是你不想，如果去做了，80%都会成功。不要给自己找不会的借口。

  2. 外国人的知识真值钱，一部书300-400RMB，团购下来也是150-200RMB，国人的不行了，好不容易写本电子版的书，能卖30RMB已经算比较高的了。

  3. 任何事情都要自己愿意静下心来学，不然效率会很低。

  4. 感觉自己以前的学习效率就是shit，学习效率是要思考的，是要自己能再现的，不是从脑子里过一遍就好了，也不是你跟着书本敲遍代码就掌握的。

  5. 虚度了几十年的人生，突然感觉时间不够了。还要再加油。

为了自己能掌握的更牢固点，决定自己还是把教程复述一遍。自己掌握了，这个钱就花得值了。

首先来张完成后的截图

俗话说的好：磨刀不误砍柴工，计划，然后实施，最后成功。建议大家都按照书上的建议，列一个ToDOList的表，至少大方向上不会偏离，而且还可以把握开发进度。

Bulls Eye 的列表基本树下：

  • 在窗体上放一个按钮，名称为：Hit Me

  • 当用户点击 “Hit Me” 按钮时，弹出用户当前的分数，这里需要牵涉到分数的计算。

  • 在窗体上放两个标签，一个是“Score”，一个是“Round”，每局完成都会根据分数和局数进行变化。

  • 在窗体上放一个滑块，值域在1..100之间。

  • 当用户点击“Hit Me”按钮时，计算滑块值。

  • 上方放置一个目标值标签，每局随机生成一个数值作为目标值。

  • 比较目标值和滑块值，然后计算分数，点击按钮后弹出提示，告诉用户分数。

  • 放置重玩按钮，点击后清空分数和局数，从新计算。

当前面的一切完成后，开始今天的第一部分，快速建立应用模型。
