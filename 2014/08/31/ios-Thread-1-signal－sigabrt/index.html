iOS开发中的 Thread 1:signal SIGABRT

发表于 2014-08-31 分类于 iOS

Thread 1:signal SIGABRT

我擦，又一个错误。❌

做一个小测试，出来这个错误。

Google，有很多种解释。其中很多都是说，其它地方的 错误，引起此类血案，但是具体情况具体对待，等于没说。

1.tap on the breakpoint tab (the 6th tab over)
2.tap on the + button in the bottom left of the window
3.tap 'add exception breakpoint'
4.tap 'done' on the popup

没有找到对应的方法，然后查看output区域，有这样一个错误提示：

this class is not key value coding-compliant for the key ＊＊＊

大概意思就是没有找到这个key对应的代码编译信息。

再根据这个提示Google，

http://stackoverflow.com/questions/3088059/this-class-is-not-key-value-coding-compliant-for-the-key

这里有个答案写到：

1. Select the object in IB and go to the 'Connections Inspector'.
2. Under 'Referencing Outlets' make sure that your object isn't still connected to the old property name... if it is, click the small 'x' to delete the reference and build again.

解决之，问题就是我的label定义了两次IBOutlet，然后删除的时候只删除了代码，但是control对应的outlet还存在，有两个outlet，但是其中一个已经不存在对应的代码了。所以运行时引起错误了。

奇怪的是，xcode检测不出来，一个outlet没有定义属性代码吗？ 为什么？

或许我的就是标题错误中的一种。
